請幫我使用 Flutter（Dart）開發一個「羽球分組管理 App」，此版本需同時完成 MVP（Phase 1）與本地資料持久化（Phase 2），並確保架構可擴充至未來加入後端。

====================
【一、核心目標】
打造一個可在球場現場使用的 App，支援：
- 隨機且公平的分組
- 輪替上場
- 玩家資料與簡單統計的本地儲存

====================
【二、玩家系統】

玩家資料包含：
- id（唯一識別）
- name（名稱）
- type（Regular 固定 / Guest 零打）
- gamesPlayed（累計上場次數）
- wins（勝場，先保留欄位）
- lastPlayedRound（最後上場輪次）
- waitingRounds（等待輪數，用於公平排場）

功能：
- 新增 / 刪除玩家
- 建立「本次活動名單」（從玩家池選擇）

====================
【三、場地與人數設定】

- 可選：
  1 個場地（4 人）
  2 個場地（8 人）
- 總人數：
  - 最多 14 人（雙場地）
  - 人數可不足

需自動處理：
- 人數 < 4 → 提示不可開始
- 人數 4~7 → 單場地運作
- 人數 ≥ 8 → 可選單/雙場地

====================
【四、分組與輪替邏輯（最重要）】

請實作一個獨立模組（match_maker.dart）：

核心規則：
1. 每輪選出上場玩家：
   - 1 場地：4 人
   - 2 場地：8 人

2. 排序優先級（關鍵）：
   - waitingRounds 高者優先
   - 若相同 → 隨機

3. 避免：
   - 同一玩家連續上場（除非人數不足）

4. 每輪結束：
   - 上場玩家：
     - waitingRounds = 0
     - gamesPlayed +1
     - lastPlayedRound 更新
   - 未上場玩家：
     - waitingRounds +1

5. 輸出：
   - currentPlaying（當前場地分組）
   - nextRound（下一輪預覽）
   - waitingList

====================
【五、資料儲存（Phase 2）】

使用本地資料庫（建議 Hive）：

需持久化：
- 玩家資料
- 歷史統計（gamesPlayed, wins）
- 上次活動狀態（可選）

要求：
- 清楚的 Repository 層（player_repository.dart）
- 資料與 UI 分離

====================
【六、UI 設計】

主要畫面：

1. 玩家管理頁
2. 活動設定頁
3. 比賽畫面（主畫面）

主畫面需顯示：
- Court 1 / Court 2（當前上場）
- Next Round（下一輪）
- Waiting List（等待區）

操作：
- 「開始排場」
- 「下一輪」
- 「重置」

需支援：
- 手機畫面（響應式簡單即可）

====================
【七、專案結構】

請使用清晰架構，例如：

/lib
  /models
  /services（或 logic）
  /repositories
  /screens
  /widgets

====================
【八、程式品質要求】

- 使用乾淨架構（Separation of Concerns）
- 分組邏輯不可寫在 UI
- 每個 class 有清楚責任
- 關鍵邏輯加註解

====================
【九、預留擴充（不要完整實作）】

請在設計中預留：
- Rating（玩家強度）
- Match History（對戰紀錄）
- 智能配對（平衡強度）

並說明：
未來如何接入後端（例如 Firebase）

====================
【十、輸出內容】

請提供：
1. 完整 Flutter 程式碼（可運行）
2. 分組演算法詳細說明
3. 資料結構設計解釋
4. 未來擴充建議（如何升級到後端）
